4. How to make a tutorial

To document the various applications and engines in the OpenEarthTools toolbox users can create tutorials. Tutorials are documents describing the functionalities of applications and engines and explain how these should be used. The following tutorials further explain how to write tutorials and include them on the wiki of openearth and in the matlab help documentation.

How to write a tutorial General information about the use of cell formatting to publish an understandable tutorial.
Publishing an individual tutorial Describes how to publish individual tutorials to examine the result.
Publish settings Various options can be set while publishing a tutorial. Controlling for example the maximum number of output lines, or whether code needs to be executed at all is described in this tutorial.
Including tutorials in the OET html documentation Tutorials can be published to html and included in the html tutorial pages on the openearthtools wiki and in the matlab help navigator. This tutorial describes how to include your own tutorial in the openearth tutorial pages.

Tests are meant to check the basic functionalities of functions in the most simple form, so:

  • Test are not meant to show off, keep the tests as simple as possible!!!
  • Do not write a test that needs user input
  • Do not write a test that needs an internet connection to run

(Tests that do need user input will be added to the ignore list for automatic testing of the toolbox)
