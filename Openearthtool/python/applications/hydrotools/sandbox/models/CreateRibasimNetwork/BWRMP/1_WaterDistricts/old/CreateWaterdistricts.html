CreateWaterdistricts 

Creates waterdistricts for a RIBASIM-network from a WFLOW ldd

Author:	Daniel Tollenaar
Email:	daniel.tollenaar@deltares.nl

Requirements:
 - Python 2.7 Anaconda or PythonXY
 - PCRaster python
 - ogr/gdal libraries

OUTPUT

NNNN = basin_name

10_MAIN_RIVER_NNNN.shp         Shape-file with main rivers_shape
11_WATERDISTRICTS_NNNN.shp     Shape-file with water districts
12_WD_OUTFLOW_NNNN.shp         Shape-file with outflow points of water districts

wflow_gauges.map               PCRaster map with gauge-locations for WFLOW
wflow_subcatch.map             PCRaster map with sub-catchments for WFLOW

SYNOPSIS

CreateWaterdistricts.py [--help] 
       [-t_srs EPSG_projection] [-order river_order] [-B basin_name]
       [-RIV rivers_shape_in] [-LDD ldd_map_in] [-WS watershed_shape_in]

DESCRIPTION

-b basin name:
(required) basin name used in output shape-files
-t_srs UTM projection:
(optional) EPSG code (e.g. EPSG:32750) of UTM projection to calculate area (only if source is not in UTM already)
-order strahler-order:
(optional) Integer value of river order used to generate ldd_rivers.map (default = 3)
-riv river-layer:
(optional) File containing rivers (default is reach.shp)
-ldd Local Drainage Direction map:
(optional) PCRaster map with local drainage direction (default is wflow_ldd.map)
-ws Watershed layer:
(optional) File containing watershed used to "clump" areas not covered in LDD. Be default only watersheds are calculated for areas included in the LDD
-debug:
(optional) if set true all generated files will be preserved

EXAMPLE

CreateWaterdistricts.py -t_srs EPSG:32750 -order 2 -B PEMALICOMAL -WS 02_DAS_PEMALICOMAL.shp
